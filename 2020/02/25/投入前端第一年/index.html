投入前端第一年

Veröffentlicht am 2020-02-25 Bearbeitet am 2020-03-07 in 生活紀錄

我是在去年正式辭職，全職投入前端領域的。當時我想，應該是半年就可以開始找工作了吧。事後證明我太樂觀了。中間遇到許多困難，我當初完全沒有預想到。一拖就是一年，好險自己還是有些存款，讓自己的生活不至於太窘迫。

事後回想起來，我覺得除了一方面是我自己或許太完美主義了，另一方面也是我自己腦袋在某些奇怪的點一直轉不過來，導致自己在學習的過程中產生許多不必要的挫折感。兩邊拉扯之下，讓自己前進的速度慢上許多。不過，這個過程中，我也學到很多。看似繞了點遠路，但收穫也還算是扎實。

一開始很簡單，後來學習曲線忽然攀高
這是當然的，因為是最入門的階段嘛! 我相信大家都是入門，先是練習用 HTML、CSS 建構出一個網頁。過了這個門檻後，要學的東西慢慢變多：響應式、jQuery、Bootstrap、javaScript、框架、打包工具、還有很多很多的名詞。不過，我得老實說，我不是依循著學校的建議，一個一個接著學習的。我在 HTML、CSS 結束後，直接開始 javaScript。當時我是想其實最後都會遇到，聽說學習上是會有些門檻，所以我就想說，不如就趁我還最有興趣的時候開始吧！雖然一般是建議 jQuery 先學，因為比較簡單 (這個是真的，我在摸索了一陣子 JS 和 Vue之後，發現 jQuery 簡直是天堂…），可是我覺得之後還是會碰到，所以就還是從 JS 開始。

迷路
其實我也不算是大迷路，因為誤走了這段路之後，我發現我對 JS 比較有概念了。只是不知道這對初學者來說，是不是有這個必要性。我在剛開始學 JS 時，被程式語言的專有名詞搞得暈頭轉向的。即便在搞懂了每個語法怎麼寫，會有什麼樣的效果，但是實際遇到問題我好像也只能照表操課，把上課學到的東西「背」下來。

我實在很不喜歡這種感覺。所以，當時花了一點時間上網找資料。後來，找到了哈佛提供的開放課程：CS50。老師用非常簡單易懂的方式，帶學生弄懂程式語言的基本概念。雖然聽得懵懵懂懂的，但也大致上為我建立了一些程式語言的基本概念：資料、變數、流程控制、pointer…等等。要我忽然解釋這些東西，我大概還沒辦法好好解釋清楚。但是，這樣跑一輪下來，確實是讓我對程式語言有個基本的認知，讓我不會因為專有名詞而亂了自己的腳步。

撞牆期後
我覺得去年下半年我算是遇到了一段撞牆期。當時我自己搞不太懂自己的問題到底是什麼，只覺得自己好像不斷在追逐我看不見的目標。找不到一個明確的方向。當時，我也知道只要好好跟著學校的步調走就好了，慢慢累積出實作經驗，很多問題應該沒這麼複雜。但是，我還是鑽牛角尖了一段時間．．．。不過，我覺得這跟「前端」這件事沒關係，是因為我想要好好釐清，我是不是要走這條路。

今年年初，我才意識到，我這樣兩邊地探索，太發散自己的力氣了。這樣下來，兩邊都做不好。我才下定決心，要把前端這部份好好完成。

或許沒有白費

在經過一段時間的迷惘和撞牆之後，明明這段時間我沒有很認真寫code，常常是看看影片、寫寫文章、東摸西摸；可是，年初在趕幾份作業時，我發現很多我以前卡住的點通通沒問題了？！不只如此，還意外地看見自己去年許多盲點。例如，我不知道為什麼就是很喜歡把高度寫死，完全沒有意識到網頁內容會自動把元素撐開；或是，我很習慣下一大堆class名稱，但是重新檢閱後，發現其實沒有那個必要性。這部分我真的滿意外的。我在想，大腦可能是需要點時間做資訊的整合吧！
